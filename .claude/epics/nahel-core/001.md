# Task: Schema layer — types, validation, injected ID/clock

## Description

Build `src/schema/`: TypeScript types + zod validators for every schema-v1 record, the shared ID generator, and the injected `Env` (clock + RNG). This is the contract layer — CONTEXT.md's glossary is normative for every name.

## Acceptance Criteria

- [ ] Zod schemas + inferred TS types for: WorkItem frontmatter (`id`, `name`, `type`, `status`, `lane`, `parent?`, `depends_on`, `external_refs`, `claimed_by?`, `created`, `updated`), Run (`id`, item ref, `actor`, `lane`, `phase`, status, `started`, `ended?`), JournalEvent (`id`, `ts`, `seq`, `type`, `actor` required, `run?`, `item?`, `payload`), Observation frontmatter (`id`, `created`, `tags`, `sources`), Config (knowledge paths, actor entry)
- [ ] Enums exactly per CONTEXT.md: type `feature|bug|chore|plan|prototype|qa`; status `backlog|in-progress|blocked|in-review|done|dropped`; lane `direct|epic-lite|full`; actor kind `human|agent`
- [ ] Core journal event-type constants defined; schema accepts unknown types (open extension)
- [ ] ID generator: 8-char lowercase base32 (unambiguous alphabet), driven by injected RNG; single generator shared by items/runs/events/segments
- [ ] `Env` object (clock, RNG) defined and threaded as the only source of time/randomness; fixed-Env tests prove identical inputs → identical outputs
- [ ] Validators reject each malformed-field case with actionable messages (verbose tests, one per field class)

## Technical Details

- Files: `src/schema/{records.ts, enums.ts, events.ts, id.ts, env.ts}`, tests in `tests/schema/`
- Deps: `zod` (add to package.json). No fs access in this layer.
- Timestamps ISO-8601 UTC strings per repo datetime rule.
