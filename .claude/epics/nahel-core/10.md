# Task: Intervention ops — pause, claim, handback

## Description

Implement PRD F9: `nahel pause <run>`, `nahel claim <item>`, `nahel handback <item>` with subtree coverage, config-resolved actor identity, and deterministic handback evidence. Enforcement itself already lives in the store's `mutate()` choke point (3) — this task adds the ops, the baseline capture, and the proof tests.

## Acceptance Criteria

- [ ] `pause <run>`: suspends an active run (hot-state status), journals the event
- [ ] `claim <item>`: sets `claimed_by` from resolved actor; covers the **entire subtree**; pauses active runs touching any covered item; journals a claim event carrying the baseline — HEAD commit SHA + `git status --porcelain` snapshot
- [ ] `handback <item>`: clears the claim; journals commits since baseline (SHAs), diff summary baseline→HEAD (files, +/-), current dirty state, and lists changes uncommitted at claim time as excluded from attribution; byte-identical evidence for identical repo state (test)
- [ ] Enforcement proof tests: agent-actor mutation refused on a claimed item AND on a descendant of a claimed ancestor (PRD success criterion 7); human actor passes; handback restores agent access
- [ ] `claim` on an already-claimed item (same or different actor) and `handback` by non-claimant: clear errors
- [ ] Paused runs are visible in `status` and `brief` pending-decisions section

## Technical Details

- Files: `src/commands/intervene.ts` (or pause/claim/handback modules), git baseline capture in `src/store/baseline.ts`, registration in `src/cli.ts`, tests in `tests/commands/intervene.test.ts` using real git temp repos
- Git evidence via child-process `git` invocations through the store layer; deterministic formatting (no locale/relative dates)
