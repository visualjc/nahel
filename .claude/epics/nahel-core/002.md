# Task: Store layer — atomic I/O, journal segments, mutation choke point

## Description

Build `src/store/`: the only layer that touches the filesystem. Owns ADR-0012's merge-safety and the PRD F1 concurrency semantics: atomic writes, journal segment ownership, deterministic merge-read ordering, the `mutate()` choke point (claim enforcement + auto-journaling), rotation, and actor resolution. This is the epic's critical path — every command is thin over it.

## Acceptance Criteria

- [ ] Layout under `nahel/` per PRD F1: config, `items/`, `runs/`, `journal/`, `observations/`; knowledge paths read from config
- [ ] Frontmatter read/write for items/observations: parse via `yaml` + ~10-line frontmatter split; writes are temp-file + atomic rename; killed-process test leaves no half-written record
- [ ] Journal append: single-line `O_APPEND` writes; per-segment monotonic `seq`; segment resolution — run events → that run's segment, non-run events → writer-scoped session segment with random ID; no two writers ever share an active segment (test: two concurrent writers, distinct files)
- [ ] Journal merge-read: streaming merge across segments ordered ts → seq → event id; identical total order on shuffled segment sets (property test)
- [ ] `mutate()` choke point: every item/run mutation goes through one function that (1) resolves actor (config `actor` entry / `NAHEL_ACTOR` env), (2) refuses agent mutations on claimed items **including descendants of a claimed ancestor**, (3) applies the write, (4) auto-appends the corresponding journal event
- [ ] Rotation: archives closed segments only (ended runs, closed sessions); active segments untouched (test)
- [ ] Hot state `state.json` read/write scoped to the run record
- [ ] Merge-safety proof at store level: two real git worktrees mutate + log in parallel; merge produces zero conflicts

## Technical Details

- Files: `src/store/{layout.ts, frontmatter.ts, journal.ts, mutate.ts, rotate.ts, actor.ts, hotstate.ts}`, tests in `tests/store/` using `fs.mkdtemp` real temp dirs and real `git` for the worktree test
- Deps: `yaml`. All time/randomness via injected `Env` from 001.
- Streaming merge must not load whole journal into memory (PRD perf: <1s at thousands of events)
