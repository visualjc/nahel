# Task: nahel log — typed events, non-run segments

## Description

Implement `nahel log` (PRD F4): append a typed journal event for *observations about work* (task started, test failed, decision made, assumption logged, …) — as opposed to mutations, which self-record. Requires actor; accepts/infers run and item refs; creates writer-scoped non-run segments as needed.

## Acceptance Criteria

- [ ] `nahel log <type> [--item <id>] [--run <id>] [--data <json|key=val>]` appends a schema-valid event to the correct segment
- [ ] Actor resolved from config/`NAHEL_ACTOR`; missing/malformed actor is a hard error with a fix-it message
- [ ] Core event types accepted by name; unknown types allowed (open extension) but flagged distinctly in output
- [ ] Non-run events land in the writer's session segment (random-ID, created on first use); run-ref'd events land in that run's segment
- [ ] Refs validated: `--item`/`--run` must exist
- [ ] Round-trip test: logged events appear in merge-read in correct total order

## Technical Details

- Files: `src/commands/log.ts`, registration in `src/cli.ts`, tests in `tests/commands/log.test.ts`
- Thin over store's `appendEvent`; no fs access here
