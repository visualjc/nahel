# Task: nahel validate — integrity checks and warnings

## Description

Implement `nahel validate` (PRD F8): schema validity of every record, referential integrity, claim-violation and claim-conflict detection, journal segment well-formedness, and rotation/compaction-overdue warnings. Exit 0 clean / non-zero on errors; warnings don't fail but surface in `brief`.

## Acceptance Criteria

- [ ] Schema validation of all items/runs/events/observations/config via 2 validators; each failure reported with file path + field + fix hint
- [ ] Referential integrity: `parent`, `depends_on`, run→item, event→run/item, observation `sources`→event IDs; dangling refs are errors; circular `parent`/`depends_on` detected
- [ ] Claim checks: journaled agent mutations on items claimed at event time = violation; conflicting claims (same item, different `claimed_by` after merge) = error
- [ ] Journal well-formedness: parseable lines, monotonic `seq` per segment, no duplicate event IDs across segments
- [ ] Journal-ahead divergence: detects records behind their latest mutation event (crash window from 3's write-ahead ordering); `validate --repair` replays pending mutations via the store's `replayPending()` — the only flag that mutates, and it only materializes what the journal already records (test: kill-injected divergence fixture → detect → repair → clean)
- [ ] Warnings (non-fatal): rotation overdue, compaction overdue (thresholds from config; per ADR-0004 these surface semantic-maintenance debt)
- [ ] Seeded-corruption test suite: each check has a fixture that trips it; clean repo validates silently exit 0
- [ ] Exposes a library interface (`validate(snapshot) → findings[]`) that `brief` (8) consumes for its warnings section

## Technical Details

- Files: `src/validate/{checks.ts, index.ts}`, `src/commands/validate.ts`, registration in `src/cli.ts`, tests in `tests/validate/`
- Checks are pure functions over the 7 snapshot + raw segment scan; command wraps and formats
