# Task: nahel brief — composition, verbatim extraction, truncation

## Description

Implement `nahel brief` (PRD F7): the deterministic onboarding pack. Composes the constitution extract, knowledge pointers, the status renderer, truncated progress renderer, pending human decisions, and validate warnings — in fixed order, under a 4 KB budget with fixed-priority truncation that is always marked. This is the primary Stage B exit artifact.

## Acceptance Criteria

- [ ] Required sections in fixed order: (1) goal + hard constraints extracted **verbatim** from PRODUCT.md by heading convention (`## Goal`, `## Hard constraints`) — never summarized; (2) knowledge pointers from config; (3) item statuses (7 renderer); (4) truncated recent activity (7 renderer); (5) pending human decisions (claims, blocked items); (6) validate warnings (9's checks — integrate behind an interface stub if 9 lands later)
- [ ] 4 KB target: truncation drops in fixed priority (oldest activity first, then done-item detail); required sections never dropped; every truncation visibly marked; over-budget constitution text truncated with explicit pointer to PRODUCT.md, never silently
- [ ] Missing PRODUCT.md or missing conventional headings → stated in the brief as a finding, not an error
- [ ] Deterministic: same state → byte-identical brief (fixed-Env test)
- [ ] PRD success criterion 1 rubric coverage test: the brief for a fixture repo contains the data needed to answer all 5 rubric points (goal, constraints, in-progress, next action, canonical truth locations)

## Technical Details

- Files: `src/views/brief.ts`, `src/commands/brief.ts`, registration in `src/cli.ts`, tests in `tests/views/brief.test.ts`
- Heading extraction: literal section slicing on the frozen template headings from 4 — no markdown AST dependency needed
- Compose 7's pure renderers; do not duplicate rendering logic
