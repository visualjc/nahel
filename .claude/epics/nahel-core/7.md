# Task: Views — nahel status and nahel progress

## Description

Implement the two read-only views (PRD F5, F6) as pure renderers over one loaded-state snapshot: `status` (work-item tree with type/status/lane, active runs + phases, claims; `--json`) and `progress` (journal timeline, newest-last, filterable). Their renderers are the machinery `brief` (8) composes — design them as importable functions, not command-locked code.

## Acceptance Criteria

- [ ] Shared snapshot loader: one store read produces the state object both views (and later `brief`) render from
- [ ] `status`: hierarchy rendered via `parent` (epics → tasks), shows type/status/lane per item, active runs with phase, `claimed_by` markers; `--json` emits the snapshot; exits non-zero on unparseable state
- [ ] `progress`: merged journal timeline, newest-last; `--item <id>` (includes descendants), `--since <iso>`, `--limit <n>`; contains nothing that isn't in the journal (strictly a view)
- [ ] Deterministic output: same state → byte-identical output (fixed-Env test)
- [ ] Performance: <1s with thousands of events (streaming, no full journal load)

## Technical Details

- Files: `src/views/{snapshot.ts, status.ts, progress.ts}`, `src/commands/{status.ts, progress.ts}`, registration in `src/cli.ts`, tests in `tests/views/`
- Renderers are pure functions `(snapshot|events) → string`; commands are I/O wrappers
