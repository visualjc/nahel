# Task: nahel init — scaffold, config, templates

## Description

Implement `nahel init` (PRD F2): non-interactive scaffold of the `nahel/` structure, config with flag-overridable defaults, and the three templates — PRODUCT.md constitution skeleton (with change-log section and the conventional headings `## Goal` / `## Hard constraints` that `brief` extracts verbatim), CONTEXT.md glossary skeleton, AGENTS.md conversational entry point.

## Acceptance Criteria

- [ ] `nahel init` in an empty git repo creates `nahel/` (config, items/, runs/, journal/, observations/) such that every other command immediately works
- [ ] Templates emitted: PRODUCT.md (constitution skeleton incl. `## Change log` section explaining the sign-off rule), CONTEXT.md, AGENTS.md; existing files at those paths are never overwritten
- [ ] Config records knowledge paths (PRODUCT.md, CONTEXT.md, ADR dir) with conventional defaults; defaults overridable by flags
- [ ] Re-run safety: second `init` no-ops or refuses with a clear message; never clobbers (test both cases: pristine and modified scaffold)
- [ ] Zero prompts — fully non-interactive (deterministic CLI)

## Technical Details

- Files: `src/commands/init.ts`, `src/templates/{product.ts, context.ts, agents.ts}`, CLI registration in `src/cli.ts`, tests in `tests/commands/init.test.ts`
- Arg parsing via `node:util` `parseArgs`; all I/O through store layer
- Template headings are the contract `brief` (007) depends on — treat wording of the two conventional headings as frozen once merged
