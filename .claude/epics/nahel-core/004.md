# Task: Mutation commands — item new/update, run start/update/end

## Description

Implement the write surface (PRD F3): `nahel item new`, `nahel item update`, `nahel run start`, `nahel run update --phase`, `nahel run end`. These make "agents never hand-edit state" satisfiable — the full item + run lifecycle drivable with zero hand-edits, every mutation self-journaling through the store's `mutate()` choke point.

## Acceptance Criteria

- [ ] `item new` (type, name, lane, `--parent`, `--depends-on`, `--external-ref`) creates a valid record with generated ID; prints the ID
- [ ] `item update <id>` mutates status/lane/parent/depends_on/external_refs with validated transitions; `updated` maintained by CLI; unknown fields/invalid enum values rejected with actionable errors
- [ ] `run start <item>` creates a Run + its `state.json`; `run update <run> --phase` updates hot state; `run end <run> <outcome>` closes the run and its journal segment
- [ ] Every mutation auto-appends the corresponding journal event with resolved actor (verify in tests by reading the journal — zero direct `log` calls needed)
- [ ] Mutations on claimed items by agent actors are refused (choke-point behavior surfaced with clear CLI error)
- [ ] Referential validation at write time: `--parent`/`--depends-on` must reference existing items

## Technical Details

- Files: `src/commands/{item.ts, run.ts}`, registration in `src/cli.ts`, tests in `tests/commands/`
- Commands stay thin: parse args → call store; no fs access in this layer
- Status transition rules: any→any within the enum is legal in v1 *except* re-opening `done|dropped` requires explicit `--reopen` flag (guard against accidental resurrection); document in `--help`
